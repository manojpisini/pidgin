pidgin // documentation

protocol overview

Pidgin is a safety-gated agent-to-agent communication protocol. Every packet is parsed, validated against schema, checked against safety rules, resolved for references, and expanded into executable YAML before delivery.

packet format

9-line .pgn packets. First line is the directive. Required fields: wf, mode, in, out. Optional: do, deny, risk, human, ttl.

@run <run_id>
wf=<workflow>
mode=draft|approval
in=[<inputs>]
out=[<outputs>]
do=[<actions>]
deny=[<denied_actions>]
risk=low|med|crit
human=yes|no

pipeline stages

01 LEXER

Tokenizes raw .pgn text. Detects directive type (@run, @log, @resolve, @emit).

02 PARSER

Extracts key-value fields. Parses array syntax [a,b,c]. Generates run_id from directive.

03 SYNTAX

Validates required fields are present. Checks field types and value ranges.

04 SCHEMA

Validates against workflow schema. Checks risk level, human flag, mode permissions.

05 SAFETY

Evaluates 9 safety rules (SG-1 through SG-9). Blocks dangerous combinations. Can require human approval.

06 RESOLVE

Resolves reference aliases. Maps short names to full paths. Checks reference existence.

07 EXPAND

Generates full YAML execution spec. Includes routing, TTL, executor recommendation.

safety rules

SG-1  deny contains actions from do
SG-2  critical risk with human=no
SG-3  publish/send in do
SG-4  delete in do
SG-5  secrets in do or deny
SG-6  file traversal in inputs
SG-7  missing human on critical
SG-8  invalid risk level
SG-9  approval mode without human

api endpoints

POST /api/v1/parse     -> parse directive, extract fields
POST /api/v1/validate  -> check required fields
POST /api/v1/check     -> safety + resolution in one call
POST /api/v1/expand    -> generate full YAML
POST /api/v1/run       -> execute full pipeline
GET  /api/v1/health    -> server status

cli commands

pgn init              create .pidgin/ config
pgn parse <file>      parse .pgn packet
pgn validate <file>   validate fields
pgn check <file>      safety + resolve
pgn expand <file>     expand to YAML
pgn run <file>        full pipeline
pgn serve             start HTTP server
pgn docs              show protocol docs
