protocol overview

Pidgin is a safety-gated agent-to-agent communication protocol. Every packet travels through a 7-stage pipeline: lexing, parsing, syntax validation, schema checks, safety rule evaluation, reference resolution, and YAML expansion.

The protocol enforces a deny-list model: agents declare what they will do and what they will deny. Safety rules block dangerous combinations before execution.

packet format

@run <run_id>
wf=<workflow>
mode=draft|approval
in=[<inputs>]
out=[<outputs>]
do=[<allowed>]
deny=[<denied>]
risk=low|med|crit
human=yes|no

required fields

  • wf workflow identifier
  • mode draft or approval
  • in input references array
  • out output references array

optional fields

  • do allowed actions
  • deny denied actions
  • risk risk level
  • human human-in-the-loop flag
  • ttl time-to-live
  • note free text annotation

handoff fields

Agent-to-agent handoff is embedded directly in the packet. The format is flat key=value — each handoff field adds one line. The 9-line convention is a readability guide, not a hard cap. Packets can contain 12-15 lines without breaking the parser.

from=agent_alpha           # source agent username
to=agent_beta             # target agent username
chain=[agent_alpha,agent_beta,agent_gamma]  # full chain
reply_to=<run_id>        # respond to previous packet
handoff=origin=<run_id>          # original packet id (preserved across chains)

handoff modes

  • forward — pass packet to next agent, no modification
  • relay — agent processes and re-wraps for next hop
  • delegate — agent hands off entire responsibility

chain limits

Safety rules enforce chain depth. Default max: 5 hops. Configurable in SAFETY_RULES.yaml. Deep chains trigger SG-10 (chain depth exceeded).

directives

  • @run execute a workflow
  • @result outcome of a task
  • @approval human sign-off
  • @context request additional context

pipeline stages

01 LEXER

Tokenizes raw .pgn text. Detects directive type. Handles whitespace normalization and encoding detection.

02 PARSER

Extracts key-value pairs. Parses array syntax. Generates deterministic run_id.

03 SYNTAX

Validates required fields present. Checks field types and value ranges.

04 SCHEMA

Validates against workflow schema. Checks risk/human/mode compatibility.

05 SAFETY

Evaluates 9 safety rules. Blocks dangerous combinations. Requires human approval when needed.

06 RESOLVE

Resolves reference aliases. Maps short names to full paths. Checks existence.

07 EXPAND

Generates full YAML execution spec. Includes routing, TTL, executor recommendation.

safety rules

SG-1  deny contains actions from do
SG-2  critical risk with human=no
SG-3  publish/send in do
SG-4  delete in do
SG-5  secrets in do or deny
SG-6  file traversal in inputs
SG-7  missing human on critical
SG-8  invalid risk level
SG-9  approval mode without human=yes
SG-10 chain depth exceeded

api endpoints

POST /api/v1/parse      parse directive, extract fields
POST /api/v1/validate   check required fields
POST /api/v1/check      safety + resolution combined
POST /api/v1/expand     generate full YAML
POST /api/v1/run        execute full pipeline
GET  /api/v1/health     server status

cli commands

pgn init                create .pidgin/ config
pgn parse <file>        parse .pgn packet
pgn validate <file>     validate fields
pgn check <file>        safety + resolution
pgn expand <file>       expand to YAML
pgn run <file>          full pipeline
pgn serve               start HTTP server
pgn docs                display docs
