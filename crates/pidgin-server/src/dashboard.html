Agent Handoff Protocol

Pidgin is a compact, local-first protocol for structured handoffs between AI agents. Nine lines of key-value text, validated through a deterministic safety pipeline.

Capabilities
What Pidgin does
✍

9-Line Packets

Directive, workflow, mode, inputs, outputs, actions, deny list, risk level, human flag. Compact enough to paste in a chat message.

🛡

Safety Gate

9 rules enforce at runtime: contradiction detection, human approval for dangerous actions, private path protection, critical-risk dual-packet approval.

🔗

Reference Resolution

Short refs like file:src/main.rs or ep:UNIT012 resolve to real paths with canonicalization, containment checks, and symlink traversal protection.

🚀

Executable Expansion

Packets expand to fully-specified YAML with resolved paths, safety metadata, risk assessment, recommended executor, and fallback chains.

📈

Token Estimation

Estimate token cost of packets vs verbose natural language. Compare a 9-line .pgn against the equivalent paragraph of text.

🔒

Fail Closed

Unknown workflow → blocked. Missing input → blocked. Ambiguous ref → blocked. If uncertain, the protocol says no.

Safety Rules
9 rules. Fail closed.
SG-1Action in both do and deny → blocked
SG-2Human-gated action without human=yes → blocked
SG-3High/critical risk forces human=yes, cannot override
SG-4References to private paths (.env, *.key, .git/) → blocked
SG-5Unknown workflow not in registry → blocked
SG-6Invalid mode for the workflow → blocked
SG-7note field is never parsed for instructions
SG-8Unresolved required inputs → expansion blocked
SG-9Critical risk requires @approval packet
Protocol
Packet grammar
# Header — directive + run ID @run agent.task_name # Key-value fields wf=generic_review mode=draft in=[primary_subject, source_refs] out=[review_notes] do=[draft, review] deny=[publish, send, delete, secrets] risk=med human=yes # Directives: @run, @result, @approval, @context # References: file:path, ep:id, workflow:name, or bare aliases # Risk: low | med | high | crit
