protocol overview

Pidgin is a safety-gated agent-to-agent communication protocol. Every packet travels through a 7-stage pipeline: lex, parse, syntax, schema, safety, resolve, expand.

The protocol enforces a deny-list model. Agents declare allowed and denied actions. Safety rules block dangerous combinations.

packet format

@run <run_id>
wf=<workflow>         mode=draft|approval
in=[<inputs>]          out=[<outputs>]
do=[<allowed>]         deny=[<denied>]
risk=low|med|crit      human=yes|no
ttl=24                 note="text"

required

  • wf, mode, in, out

optional

  • do, deny, risk, human, ttl, note

handoff fields

from=agent_alpha        to=agent_beta
chain=[alpha,beta,gamma]
handoff=forward|relay|delegate
reply_to=<run_id>      origin=<run_id>

modes

  • forward - pass unchanged
  • relay - process and re-wrap
  • delegate - full transfer

context transfer

context_file=[file:path/to/doc.md]
context_id=ctx_abc123
context="inline text"
context_b64=SGVsbG8...

methods

  • context_file - file references
  • context_id - pointer to store
  • context - inline string
  • context_b64 - base64 binary

limits

  • inline: max 4KB
  • context_file: unlimited
  • context_id: unlimited
  • context_b64: max 1MB

directives

  • @run execute workflow
  • @result task outcome
  • @approval human sign-off
  • @context request context

pipeline

01 LEXER

Tokenizes raw text, detects directive.

02 PARSER

Extracts key-value, parses arrays.

03 SYNTAX

Validates required fields, types.

04 SCHEMA

Workflow schema validation.

05 SAFETY

Evaluates 10 safety rules.

06 RESOLVE

Resolves reference aliases.

07 EXPAND

Generates YAML execution spec.

safety rules

SG-1   deny contains actions from do
SG-2   critical risk with human=no
SG-3   publish/send in do
SG-4   delete in do
SG-5   secrets in do or deny
SG-6   file traversal in inputs
SG-7   missing human on critical
SG-8   invalid risk level
SG-9   approval mode without human=yes
SG-10  chain depth exceeded

cli
